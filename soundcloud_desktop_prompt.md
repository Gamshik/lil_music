## 🎯 Цель проекта

Создать полноценный desktop-клиент, который:

- выглядит как современное приложение (не "веб-страница в окне")

- имеет удобный UX
- устойчив к ошибкам API

Важно: это **не клон SoundCloud**, а **альтернативный клиент с собственным UX и локальными возможностями**.

## ⚠️ Ограничения проекта

На текущем этапе:

- ❌ Нет OAuth (нет пользовательской авторизации)
- ❌ Нет доступа к приватным данным SoundCloud
- ❌ Используется "серый" client_id (временное решение)

Это значит:
- работаем только с публичными данными
- делаем локальную логику пользователя (лайки, плейлисты)

## 🧩 Основной функционал

### 🔊 Работа с треками
- поиск треков через API
- получение метаданных
- воспроизведение аудио
- управление playback (play/pause/seek/volume)
- очередь воспроизведения

---

### 📚 Локальная библиотека (ключевая часть)

Так как нет OAuth — все пользовательские данные реализуются локально:

- избранное (favorites)
- пользовательские плейлисты
- история прослушивания
- сохранённые треки

Важно:
Это НЕ данные SoundCloud — это данные приложения.

---

### 🖥 Desktop-функции

Приложение должно ощущаться как нативное:

- управление окном
- system tray
- горячие клавиши (media keys)
- восстановление состояния
- (опционально) мини-плеер

---

## 🏗 Архитектура (обязательно соблюдать)

Архитектура должна быть чётко разделена:

```
UI (Web Layer)
↓
Bridge (JS ↔ C++)
↓
Application/Core Logic
↓
Infrastructure (HTTP, DB, Audio)
```

### Пояснение:

- UI ничего не знает о C++
- Core не зависит от UI
- Infrastructure изолирован
- Bridge — единственная точка связи
